# Test Feature Report: clean benchmark workflow with Torch C++

- Date: 2026-05-24
- Skill: agentic-tests/test-feature
- Scenario: a downstream user starts from removed extern checkouts, runs the top-level benchmark workflow, and expects Torch C++ results in generated benchmark reports.
- Command under test:
  `BENCH_INSTANCE=bin_matmul_256 BENCH_RUNS=1 BENCH_WARMUPS=0 PUBLICATION_GATE_SUITE=small ./scripts/run_all.sh 1`

## Verdict

Pass after fixes.

The workflow can clone `extern/tenferro-rs` and `extern/pytorch-openblas`, build PyTorch/LibTorch v2.12.0 linked to Homebrew OpenBLAS, run the top-level suite, and generate:

- `result/einsum-results.md`
- `result/cpu-benchmark-results.md`

## Issue Found

`scripts/run_cpu_ops.sh` only converted tenferro `publication_gate` rows into the PR884 CPU comparison CSV. The formatted table had columns for Torch C++, PyTorch Python, and JAX Python, but those columns were rendered as `-` because no backend runners were invoked for CPU-op items.

## Fixes Applied

- Added `cpp/benchmark_cpu_ops_libtorch.cpp` and the CMake target `benchmark_cpu_ops_libtorch`.
- Added `scripts/benchmark_cpu_ops_python.py` for PyTorch and JAX CPU-op rows.
- Updated `scripts/run_cpu_ops.sh` to append Torch C++, PyTorch Python, and JAX Python measurements to `data/results/cpu_ops_t<threads>_<timestamp>.csv`.
- Updated README, AGENTS.md, and docs to describe the supported CPU-op backend behavior.
